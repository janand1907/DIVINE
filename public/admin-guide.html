Kanban View

Shows columns for each status: New → Contacted → Quoted → Negotiation → Confirmed → Lost.

  • Drag a card to a different column to update its status instantly.
  • A note is automatically added to the lead's timeline when you move it.
  • Click any card to open the full detail drawer.

List View

A sortable table with columns: Name, Mobile, Destination, Status, Priority, Assigned To, Follow-up Date, Created.

  • Use the Search box to filter by name, mobile, or destination.
  • Use the Status dropdown to filter by a single status.
  • Click any row to open the detail drawer.

🧳 Packages

Packages are your tour products shown on the website. Go to Content → Packages.

Package List

Shows all packages with title, slug, category, destination, price, status (published/draft), and featured flag. Use the New Package button to create one.

ℹ️
Slug is the URL path — e.g. slug kedarnath-yatra-5n-6d makes the page live at /packages/kedarnath-yatra-5n-6d. It must be unique and use only lowercase letters, numbers, and hyphens.
⚠️
A package only appears on the website when Published is toggled on. Unpublished packages are invisible to visitors but fully editable.

🔍 SEO Pages

Override SEO metadata for specific pages of the site. Go to Site → SEO Pages.

How it works

Each row targets a specific URL path (e.g. /about, /contact, /divine-tours). The metadata you enter here overrides the default site-wide values for that page only.

FieldNotes
Path *The URL path, e.g. /about or /divine-tours
SEO TitleBrowser tab title and Google search headline (50–60 characters ideal)
SEO DescriptionGoogle snippet description (150–160 characters ideal)
OG ImageImage URL shown when the page is shared on WhatsApp / social media
Canonical PathOnly needed if this page has duplicate content elsewhere
IndexToggle off to tell Google NOT to index this page (e.g. thank-you pages)

Dynamic Vehicle Pricing

For each route, set up a pricing table showing different vehicle types and their prices:

  • Vehicle Type: e.g. "Economy", "Comfort", "Luxury"
  • Seats: Number of passengers the vehicle accommodates
  • Price: Transfer price for this vehicle type on this route

Add or remove pricing rows as needed. The table updates immediately on the frontend.

Frontend

  • All active routes appear on /airport-transfers.
  • Each route has a detail page at /airport-transfers/[slug] with pricing and booking info.
💡
Use consistent vehicle type names across routes (e.g. always "Economy", "Comfort", "Luxury") so customers understand pricing tiers.

🏨 Hotel Assistance

Hotel assistance requests are freeform inquiries submitted through the website. No dedicated admin panel is needed.

How it works

  • Hotel assistance form submissions automatically flow into the Leads section.
  • Each submission has a source = "hotel-assistance" tag.

Viewing Hotel Requests

Go to Content → Leads and filter by source to see hotel assistance inquiries.

Hotel Lead Data

Each hotel assistance lead includes these custom fields:

FieldNotes

Hotel requests share the same pipeline as other leads — manage them using the standard Leads view with all status and assignment tools.

📜 Activity Log

A read-only audit trail of all admin actions. Go to Overview → Activity Log.

Every create, update, and delete action is logged with the user email, timestamp, action type, and the entity affected. Use this to track who changed what and when.

💡 Tips & Shortcuts

Common Workflows

  1. Adding a new tour package
    Create category and destination first (Taxonomies) → then create the package → upload images to Media Library → copy URLs into the Package gallery fields → toggle Published on.
  2. Handling a new lead
    Dashboard → click the lead → change Status to "Contacted" → set Follow-up Date → assign to a team member → add a note → Save.
  3. Publishing a blog post
    Blogs → New Post → fill Title, Slug, Content → add Cover Image → toggle Published on → Save.
  4. Updating contact details site-wide
    Theme & Branding → update WhatsApp Number, Contact Phone, Contact Email, Address → Save.
  5. Connecting Google Analytics
    Site Settings → paste your GA4 ID (e.g. G-XXXXXXXXXX) → Save. Analytics starts tracking immediately on next page load.
